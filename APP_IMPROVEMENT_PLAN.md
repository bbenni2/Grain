## 0. Context — what this repo is

- **Grain** = local-first macOS photo pipeline. Everything runs **locally via Ollama**
  (no cloud, no Anthropic API in the shipped product — the `ai.provider: claude`
  path in `config.yaml` is a dormant fallback, NOT used).
- Website is **done & deployed** to `getgrained.app` (repo `bbenni2/Grain`, GitHub
  Pages from `main` root). Source lives in `photo-pipeline/website/`.
- The desktop app is **`app_qt.py`** (PyQt6, ~2560 lines, single file).
- Pipeline logic is in `pipeline/*.py`, orchestrated by `main.py::run_pipeline()`.

### Do NOT edit (build artifacts — regenerated by build):
- `Grain.app/`, `dist/`, `build/`, `.venv*/` — copies of source. Only edit the
  top-level source: `app_qt.py`, `main.py`, `pipeline/*.py`, `config.yaml`, `website/`.

### How to run the app (testing is ALLOWED — user approved launching it):
```bash
cd "<repo>/photo-pipeline"
.venv-arm64/bin/python app_qt.py      # Python 3.13 arm64 venv with PyQt6
```
- To verify UI visually: launch via computer-use (open Terminal, run the command),
  then screenshot. Or `osascript`/`open` the built `Grain.app`.
- For pure logic changes: `.venv-arm64/bin/python -m py_compile <file>` + small
  ad-hoc test scripts (see Section 5 pattern).

## 3. Cross-cutting gotchas / rules

- **`_write_config_values` is now section-aware** — when persisting a key that exists in
  multiple YAML sections (`enabled`, etc.), pass `"section.key"`. For globally-unique
  keys a bare key still works.
- **`_save_settings`** (app_qt.py ~1114) uses its own `_set`/`_set_in` helpers — keep
  those in sync if you add settings.
- **PyInstaller** (`grain.spec`): any new asset (fonts, noise PNG, icons) MUST be added
  to `datas=[...]` or it won't ship. Test BOTH `python app_qt.py` AND the built app.
- **Don't break the CLI**: `main.py` and `dex_pipeline.py` share `pipeline/`. UI work
  shouldn't change pipeline function signatures without updating all callers
  (`grep -rn "process_brackets\|cull(\|ingest(" main.py dex_pipeline.py`).
- **Keep motion calm** (website ethos): 200–400ms, no bouncy springs.

## 5. Bracket logic test (recreate if the spawned task didn't land)

Save as `tests/test_bracket.py`, run with `.venv-arm64/bin/python -m pytest tests/ -q`
(or run directly). Construct `PhotoRecord`s, set `.filename`, `.datetime_original`
("YYYY-MM-DD HH:MM:SS"), `._shutter` (float sec), `.ev_bias`, `.local_scores =
LocalScores(); .composite`. Assert:
- `_detect_fixed_bracket_groups(recs, 3)` → two groups when there's a >15s gap between
  two 3-frame bursts.
- `_assign_bracket_roles`: EV mode (-2/0/+2) → BASE=0EV, UNDER=-2, OVER=+2.
  Shutter fallback (ev all 0, shutters differ) → BASE=median shutter, longer=OVER,
  shorter=UNDER. **Enum values are UPPERCASE** ("BASE","UNDER","OVER").
- `_detect_bracket_groups(recs)` auto → 3-frame burst with shutter ratio >1.6 = 1 group.
